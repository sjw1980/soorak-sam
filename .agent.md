## Traceability Rules (권장)

- **요구사항 ID 표기**: 모든 관련 산출물(설계, 코드, 테스트, PR, 커밋)은 `[REQ-<번호>]` 형식으로 참조
- **아티팩트 메타데이터**: 문서 상단에 `Artifact-ID: ART-<type>-<YYYYMMDD>-<seq>` 표기
- **PR/커밋 메시지 포맷**: 첫 줄에 `SWE<n>: [REQ-123] 설명` 형식 권장
- **매핑표 유지**: `traceability/trace_matrix.csv`(또는 `traceability/trace_matrix.md`)에 요구사항↔아티팩트 매핑을 자동 업데이트 권장
- **검증 증빙**: 테스트 결과, 리뷰 로그, 정적분석 리포트는 `artifacts/verification/` 아래에 보관하고 매핑표에 링크

## Templates / Artefact Conventions (예시)

- PR 템플릿(요약):
  - `Related REQs: [REQ-123, REQ-124]`
  - `Affected modules:`
  - `Traceability updates (file): traceability/trace_matrix.csv`
- Commit 메시지 예: `SWE-3: [REQ-123] Implement feature X (module: src/components/x)`
