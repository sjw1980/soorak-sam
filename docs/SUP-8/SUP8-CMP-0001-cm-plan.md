## 1. 개요

### 1.1 목적

본 형상 관리 계획(CMP)은 CLI 계산기 소프트웨어 프로젝트의 산출물·소스코드·빌드 환경·테스트 결과 등 모든 형상 항목(CI)을 체계적으로 식별·통제·감사하기 위한 절차와 책임을 정의한다.

### 1.2 범위

| 포함 | 제외 |
|------|------|
| SWE-1~SWE-6 산출물 문서 | HW 형상 항목 |
| SPL-2 빌드 환경·피처·출시 산출물 | 외부 라이브러리 소스코드 |
| SUP-1 QA 산출물 | Google Test 소스 (FetchContent 관리) |
| SUP-8 형상 관리 산출물 자체 | CI/CD 파이프라인 설정 |
| 소스코드 (src/) | |
| 빌드 스크립트 (CMakeLists.txt) | |

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| CI (Configuration Item) | 형상 관리 대상으로 식별된 개별 산출물·파일·문서 |
| BL (Baseline) | 특정 시점의 승인된 CI 집합 — 이후 변경은 CR을 통해서만 가능 |
| CR (Change Request) | 베이스라인 이후 CI 변경을 위한 공식 요청 |
| CCB (Change Control Board) | CR 승인·거부를 결정하는 변경 제어 위원회 |
| CSA (Configuration Status Accounting) | CI의 현재 버전·상태를 기록·보고하는 활동 |

## 6. 변경 제어 절차

```
[변경 요청 제출] → [영향 분석] → [CCB 검토·결정] → [구현] → [검증] → [베이스라인 갱신]
```

| 단계 | 활동 | 담당자 | 산출물 |
|------|------|--------|--------|
| 1. 요청 | CR 작성 (SUP8-CR-XXXX) | 변경 요청자 | CR 문서 |
| 2. 영향 분석 | 영향 받는 CI 식별, 일정·비용 영향 평가 | 담당 에이전트 | 영향 분석 결과 |
| 3. CCB 결정 | 승인 / 거부 / 보류 결정 | CCB | CCB 결정 기록 |
| 4. 구현 | 승인된 CR에 따른 CI 변경 수행 | 담당 에이전트 | 변경된 CI (신 버전) |
| 5. 검증 | 변경 결과 검증 (테스트 재수행 등) | QA / 담당 에이전트 | 검증 결과 |
| 6. BL 갱신 | 검증 완료 후 베이스라인 갱신 등록 | SUP-8 담당 | 갱신된 BL 목록 |
