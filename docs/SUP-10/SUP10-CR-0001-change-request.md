### SUP10-CR-0001 — `include/` 폴더 미활용 — 문서 내 경로 불일치 수정

| 항목       | 내용                                              |
| ---------- | ------------------------------------------------- |
| ID         | SUP10-CR-0001                                     |
| 제목       | `include/` 폴더 미활용 — 문서 내 경로 불일치 수정 |
| 변경 유형  | 문서 정정 (코드 변경 없음)                        |
| 연계 PR ID | SUP9-PR-0001                                      |
| 요청자     | ASPICE SUP-10 전문가                              |
| 요청 날짜  | 2026-04-12                                        |
| 우선순위   | P2                                                |
| 상태       | **Closed**                                        |

**변경 배경 (Background)**:

프로젝트 설계 초기에 `include/` 폴더를 퍼블릭 헤더 배치 위치로 계획하고,`src/` 하위를 `src/calc_lib/`(라이브러리 소스)와 `src/cli/`(CLI 진입점)로 분리하는 구조를 산출물에 기재하였다.
그러나 실제 구현에서는 모든 소스 파일(헤더 포함)이 `src/` 단일 디렉터리에 배치되었으며, `include/` 폴더는 `.gitkeep` 파일만 존재하는 빈 상태로 남게 되었다.
이로 인해 다수의 산출물 내 파일 경로 기술이 실제와 일치하지 않는 불일치가 발생하였다.

**변경 전/후 비교**:

| 문서                   | 변경 위치            | 변경 전 (계획)                                               | 변경 후 (실제)                                                                                      |
| ---------------------- | -------------------- | ------------------------------------------------------------ | --------------------------------------------------------------------------------------------------- |
| SPL2-BUILD-0001 §1.2   | 빌드 디렉터리 구조   | `include/`, `src/calc_lib/`, `src/cli/`                      | `include/` (.gitkeep 보관), `src/` (모든 소스 단일 위치)                                            |
| SPL2-BUILD-0001 §2.1   | SPL2-BUILD-0002 입력 | `src/calc_lib/*.cpp`, `include/*.h`                          | `src/*.cpp`, `src/*.h`                                                                              |
| SUP8-CI-LIST-0001 §1.6 | CI-0016~0019 경로    | `src/calc_lib/Calculator.h`, `src/calc_lib/InputParser.h` 등 | `src/Calculator.h`, `src/InputParser.h` 등                                                          |
| SUP8-CI-LIST-0001 §1.6 | CI-0020~0021 경로    | `src/cli/AppController.h`, `src/cli/AppController.cpp`       | `src/AppController.h`, `src/AppController.cpp`                                                      |
| SUP8-CI-LIST-0001 §1.6 | CI-0022 경로         | `include/types.h (예정)`                                     | `src/types.h`                                                                                       |
| SUP8-CI-LIST-0001 §1.6 | CI-0023 경로         | `src/cli/main.cpp (예정)`                                    | `src/main.cpp`                                                                                      |
| MAN3-PP-0001 §9        | WBS-0004 경로        | `src/calculator.cpp`, `include/calculator.hpp`, `main.cpp`   | `src/Calculator.cpp`, `src/InputParser.cpp`, `src/AppController.cpp`, `src/types.h`, `src/main.cpp` |
| SUP9-PRM-0001 §3.1     | PR-0001 해결 결과    | `include/` 미활용 언급 없음                                  | `include/` 미활용 명시적 기재                                                                       |

**변경 영향 산출물**:

- `docs/SPL-2/SPL2-BUILD-0001-build-environment.md` (SUP10-IA-0001)
- `docs/SUP-8/SUP8-CI-LIST-0001-ci-baseline.md` (SUP10-IA-0002)
- `docs/MAN-3/MAN3-PP-0001-project-plan.md` (SUP10-IA-0003)
- `docs/SUP-9/SUP9-PRM-0001-problem-resolution.md` (SUP10-IA-0004)

**비고**: 코드 변경(소스 파일 이동, `include/` 신규 배치 등)은 이번 변경 요청의 범위에 포함되지 않는다. 현재 빌드·테스트 모두 정상 동작이 확인되었으므로 문서를 실제 구조에 맞게 정정하는 것으로 이슈를 종결한다.
